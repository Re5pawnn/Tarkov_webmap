# 塔科夫地图定位工具

这是一个本地运行的《Escape from Tarkov》辅助地图工具。它通过读取游戏截图文件名中的坐标信息和本地 Log 文件，帮助你在网页地图上定位当前位置、查看战局状态、浏览商人任务链接，并支持队友位置同步。

工具只读取本地截图和日志文件，不注入游戏进程，不修改游戏内存。

## 主要功能

- 截图定位：读取塔科夫原生截图文件名中的坐标和朝向，在地图上显示当前位置与方向。
- 自动选图：根据截图坐标和 Log 识别到的地图自动切换地图，也可以手动选择地图。
- 地图点位：显示撤离点、PMC 出生点、Scav 出生点、Boss 刷新、锁门、开关、危险区、固定武器、BTR 停靠点、转移点等信息。
- 战局信息栏：读取本地 Log，展示当前地图、模式、Raid 状态、Raid ID、服务器、排队耗时、加载耗时和最近事件。
- 商人任务页：内置商人任务列表，支持播放 B 站播放器链接，也支持打开外部任务链接。
- 队友同步：支持房主模式和连接模式，在同一地图上显示队友位置、名称和颜色。
- 目录记忆：截图目录、Log 目录和同步配置会保存在浏览器本地，下次打开时自动尝试恢复。

## 运行环境

- Windows 10/11。
- 推荐使用 Microsoft Edge 或 Chrome。
- 浏览器必须支持 File System Access API，否则无法选择本地目录。
- 如果使用源码运行，需要 Python 3.10 或更高版本。
- 如果使用安装版或带 `runtime` 的版本，一般不需要单独安装 Python。

## 启动方式

### 安装版

安装完成后，使用桌面或开始菜单中的快捷方式启动。

### 源码版

双击运行：

```bat
start_tool.bat
```

也可以在当前目录手动运行：

```bat
python launcher.py
```

默认会打开：

```text
http://127.0.0.1:5173/index.html
```

如果 `5173` 端口被占用，启动器会自动选择后续可用端口。请以控制台输出的 `URL:` 为准。

## 快速开始

1. 启动工具，并保持启动器窗口不要关闭。
2. 点击“选择截图目录”，选择塔科夫截图目录。
3. 点击“选择Log目录”，选择游戏 Log 目录，想只用截图定位也可以跳过这一步。
4. 点击“立即扫描”，读取已有截图和日志。
5. 点击“开始监听”，工具会按设定间隔持续扫描新增截图和 Log。
6. 在游戏中使用塔科夫原生截图键截图，网页地图会更新当前位置。

常见截图目录：

```text
C:\Users\<你的用户名>\Documents\Escape from Tarkov\Screenshots
```

常见 Log 目录：

```text
D:\SteamLibrary\steamapps\common\Escape from Tarkov\build\Logs
```

选择 Log 时，可以选择 `Logs` 目录本身，也可以选择包含 `Logs` 的游戏目录。工具会自动寻找最新的 `log_*` 子目录。

## 地图定位

地图页会读取最新识别到的截图，并显示：

- 当前地图。
- 玩家位置。
- 玩家朝向。
- 地图点位图层。
- 队友位置，前提是已开启队友同步。

地图下拉框默认是“自动选择”。如果你手动选择了某张地图，工具会优先使用手动选择；切回“自动选择”后，会继续根据截图坐标和 Log 信息判断地图。

地图支持鼠标滚轮缩放和拖拽移动。右上方图层开关可以控制不同点位类型是否显示。

## 战局信息栏

地图下方的战局信息栏来自本地 Log 文件。它会优先展示当前战局总览，而不是完整日志。

摘要区显示：

- 监听状态。
- 当前地图。
- 战局模式。
- Raid 状态。
- 最近日志时间。

展开详情后显示：

- 当前战局：地图、模式、Raid ID、服务器、状态。
- 关键计时：最近日志、排队耗时、加载耗时、进入战局时间、结束时间。
- 最近事件：最多显示最近 5 条摘要事件。

当前可识别的事件包括：

- 会话模式。
- 角色选择。
- 战局创建。
- 地图载入。
- 匹配完成。
- 游戏开始中。
- 已进入战局。
- 匹配取消。
- 战局结束。
- 任务开始、失败、完成。
- 跳蚤市场售出或过期。

如果没有选择 Log 目录，战局栏会显示“未接入日志”或“仅截图定位”，地图定位功能仍可正常使用。

## 商人任务页

顶部切换到“任务”页后，可以查看内置商人任务列表。

- 点击商人任务名后，右侧会显示对应内容。
- B 站播放器链接会直接嵌入播放。
- 普通网页链接会以外链卡片形式打开，避免普通视频页无法嵌入导致空白。

当前任务数据是内置在工具中的任务链接数据。后续如果扩展为外部任务文件，可以按“任务名 + 链接”的形式维护。

## 队友同步

队友同步用于在同一张地图上显示队友位置。它依赖本地启动器提供的同步服务。

### 房主模式

房主开启同步后，本机会监听默认同步端口：

```text
39247
```

如果队友不在同一局域网，通常需要配合 FRP、端口转发或其他穿透方式，让队友可以访问房主的同步地址和端口。

### 连接模式

连接模式用于连接房主。

需要填写：

- 远端地址，例如 FRP 域名或房主可访问 IP。
- 远端端口，默认 `39247`。

如果连接模式没有填写远端地址，工具不会误开房主同步，会提示需要填写远端地址。

### 显示效果

同步成功后，地图上会显示队友：

- 名称。
- 颜色。
- 位置。
- 朝向。

只有当队友和你处在同一张地图，并且对方也有有效截图定位数据时，队友标记才会显示在当前地图上。

## 目录权限与本地保存

浏览器会要求你授权读取截图目录和 Log 目录。工具会把目录句柄保存在浏览器 IndexedDB 中，下次打开时会尝试恢复。

如果浏览器权限失效、清理了站点数据，或更换了浏览器，需要重新选择目录。

队友同步名称、颜色、模式、远端地址等配置保存在浏览器 localStorage 中。

## 常见问题

### 浏览器没有自动打开

查看启动器窗口中的 `URL:`，手动复制到 Edge 或 Chrome 打开。

### 提示浏览器不支持目录选择

请使用新版 Edge 或 Chrome，不建议使用 Firefox 打开本工具。

### 选择 Log 目录提示无效

请确认选择的是以下两类目录之一：

- `...\Escape from Tarkov\build\Logs`
- 包含 `Logs` 子目录的游戏目录

并确认 `Logs` 下已经有 `log_*` 子目录。刚安装或从未启动过游戏时，可能还没有可读取的日志。

### 地图没有定位

请确认：

- 已选择正确的截图目录。
- 使用的是塔科夫原生截图。
- 截图文件名包含坐标和朝向信息。
- 点击过“立即扫描”或已经开启监听。

### 战局信息没有更新

请确认：

- 已选择正确的 Log 目录。
- 已开启监听，或手动点击“立即扫描”。
- 游戏已经产生新的 `application` 或 `notifications` 日志。
- 当前 Log 格式没有被游戏版本更新破坏。

未知日志行会被跳过，不会中断整个监听流程。

### 队友同步无法开启

请检查：

- 房主端 `39247` 端口是否被占用。
- 防火墙是否允许访问。
- 连接模式是否填写了远端地址。
- FRP 或端口转发是否已经正确配置。

如果端口被占用，界面会显示类似“房主同步端口 39247 启动失败”的错误。

## 文件说明

- `start_tool.bat`：Windows 启动脚本，优先使用内置 Python，失败时尝试系统 Python。
- `launcher.py`：本地 HTTP 启动器和队友同步服务。
- `index.html`：页面结构。
- `app.js`：地图定位、日志解析、任务页、队友同步等主要逻辑。
- `styles.css`：页面样式。
- `maps_detail.json`：地图和点位数据。
- `maps_list.json`：地图列表数据。
